## 目标
- 移动端（<768px）进入站点仅显示总目录（列表），不展示正文。
- 点击列表项进入阅读视图：左上角显示“返回”按钮，点击返回到总目录；阅读视图中不显示目录浮层。
- 在阅读视图底部增加“上一篇 / 下一篇”跳转组件（显示相邻文章标题，样式参考 VitePress）。
- 刷新保留状态：使用 hash 路由 `#list` 与 `#post/<filename>`。

## 交互与路由
- 初始：若 `#post/<filename>` 则进入阅读视图并加载文章；否则进入列表视图 `#list`。
- 列表→阅读：设置 `uiMode='reader'`，写入 `#post/<filename>`，滚动置顶。
- 阅读→返回：设置 `uiMode='list'`，写入 `#list`，恢复列表滚动位置。
- 上一/下一篇：计算相邻索引，更新 hash 并加载文章；越界禁用按钮。

## 技术实现
- index.html：
  - 新增移动端阅读视图顶部“返回”按钮容器 `<button id="mobile-back">返回</button>`。
  - 在文章底部新增 `<div id="mobile-pager">` 包含“上一篇/下一篇”两块卡片（显示标题）。
- style.css：
  - 新增 `.mode-list` / `.mode-reader` 两种模式样式（移动端下列表全屏、阅读全屏）。
  - 隐藏目录浮层与“列表按钮”在阅读视图；固定顶部返回按钮与底部分页组件（适配安全区）。
  - 分页卡片样式参考 VitePress：左右排布、卡片显示相邻标题、轻阴影与手绘虚线。
- app.js：
  - 新增 `uiMode`、`setMode()` 与 `applyMode()`；管理视图切换与 CSS 类。
  - 更新 `load()` / `openPost()` / 返回逻辑与 hash 解析。
  - 实现 `openPrev()` / `openNext()` 与相邻文章预取。

## 验证
- 真机模拟小屏：进入首页只见目录；点开文章进入阅读；阅读下方有上一/下一篇卡片；点击返回回到目录。
- 刷新：`#post/<filename>` 直达阅读；`#list` 显示目录。
- 禁用越界：首篇不可“上一篇”，末篇不可“下一篇”。
